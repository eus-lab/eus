# IoT システム研究室(中本研究室)
## 研究テーマ
IoTは，モノがインターネットを介してクラウドとサーバに接続されるITシステムです． IoTシステムのアプリケーションは，人が使用するスマートフォンからデータを収集するセンサーまで，自動車などの制御と情報提供を統合するより複雑なシステムにまで及びます． 物理空間につながってモノからデータを収集し，デバイス，エッジコンピューター，クラウドでこのデータを分析し，適切にモノを制御します．また人が使うデバイスの場合はデバイスを介して集めたデータを分析，その結果に基づいて人にサービスを提供します。また，システムの観点から見ると，IoTシステムは多くのコンピュータが世界中に展開されたりするような大規模システムで，各コンピュータが協調しながら動作する分散システムです． さらに，これらのコンピュータはどこにでも存在するという意味でユビキタスシステムでもあります．

これらのコンピュータは，モノ，マシン，およびデバイスに組み込まれているため，組込みシステムとも呼ばれます． 組込みシステムは専用コンピュータで，物理空間からの入力が取得されて物理空間に出力される限られた期間内に応答するリアルタイム特性， CPUやメモリ容量，バッテリーなどのシステムリソースに対する制限がある等さまざまな特性があります． さらに，組み込みシステムには，このような制限があるため，サービスの実現にはより高度な技術が必要です．

このようなシステムは，社会を安全かつセキュアにする，社会を変革するための基礎となるシステムとなります． 私たちの研究室では，IoTシステムや組込みシステムなどのコンピュータシステムに関連するさまざまなシステムとアプリケーションソフトウェアを研究しています。現在、以下の研究テーマに取り組んでいます。

## 機械学習・AI技術のコンピューティングシステムへの応用技術の研究
機械学習はさまざまな分野で使用され始めており，IoTシステムおよび組み込みシステムの分野で期待されています． しかし，これらのシステムは，遠隔に配置されること，リアルタイム性能，厳しいリソース制約があります．これまで，機械学習は大規模な資源を有するクラウドで実行されてきましたが、IoT組み込みシステムでの利用に多くの問題があります。これらの問題を解決するために，現在次のプロジェクトに取り組んでいます．
- 転移学習を利用した利用環境変化へ適合可能な組込みシステム向け機械学習ソフトウェアの研究


## データ分析・機械学習時代のソフトウェア開発手法の研究

データ分析・機械学習時代のプログラムは，プログラムを動作させアプリケーションとして実行する目的のためのソフトウェア開発手法とは異なっているように思えます．何が本質的に違うのかを考察し、よりよりデータ分析・機械学習のためのソフトウェア開発手法をプログラミング手法やソフトウェアエンジニアリングの視点から研究します．

## 生成AIを利用した教育システムの研究

生成AIは個人へのカスタマイズが可能な対話環境を生み出します。これを利用して、学生の誤りの特性や関心に応じた教育システムを研究します。

## これまでの研究

### リアルタイムシステム系
- Cloudia: リアルタイムデータストリームシステム
	- 周りの環境からストリームデータが自動車に到着すると捉えリアルタイムにストリームをスケジューリングするアルゴリズム(情処学会論文誌 データベース 優秀論文賞)
- FISE: 分散組込みソフトウェア開発のための仮想ソフトウェア実行環境
	- 仮想マシン生成のQEMUを利用した高速CPUシミュレータ
- QoS、低消費電力を指向したCPU・センサーリアルタイムスケジューリングアルゴリズム
	- タスクの公平なQoSを実現するために制御理論を使ったリアルタイムスケジュングアルゴリズム(マツダ研究助成賞)
- センサーリアルタイムスケジューリングアルゴリズム
	- センサーネットワークを可能な限り生かすセンサーデータルーティングアルゴリズム

### モバイルシステム系
- Hyrax: 利用者の好みにカスタマイズ可能な携帯端末インタフェース
- RelSeeker:　グラフデータベースでのグラフの関係に着目したモバイルミドルウェア

### 機械学習応用
- N3:FPGAを用いた組込みシステム向け深層学習フレームワーク
- 機械学習を利用した遠隔音声配信システム
- 自然言語処理技術を使用したデータベース再構築支援システム
